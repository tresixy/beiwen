玩家进入游戏之后选择地块，然后开始冒险。每一次进入冒险都检查玩家目前所处的时代，然后两张表格的顺序弹event加入冒险，玩家可以通过合成手牌来解决events，以下是参考：

排序,时代,events名称 ,key (Key Card Solution),提示语 (Flavor Text),沙盘奖励 (Sandbox Reward),时代开场语
1,生存时代,【寒冷】,【火】,“夜幕降临，体温与希望一同流逝。”,篝火,
2,生存时代,【饥饿】,【农业】,“人口渐增，林间的野果与奔跑的野兽已不足以果腹。”,麦田,

比如玩家还在生存时代，那么进入游戏后就从几个生存时代里随机加一个events到游戏中，比如触发的为：寒冷，那么寒冷就在开始冒险之后的 合成页面的最上面，替换掉现在的 “回合”，然后设置通关条件为：使用火的手牌拖到“寒冷”的上面，就触发通关动画。

所有的钥匙卡都是红色的，类型为 Ruby,所有的灵感卡 都是白色common, 其他的非设定也就是玩家通过游玩生成的卡都是蓝色

然后，玩游戏的 时候，玩家拖入两张牌到熔炉里，触发后端AI 带着卡信息的AI 提示词发送给AI ，通过API 调用收回结果，是一个json 格式的内容：

```
[
  {
    "name": "热石",
    "prompt": "一块被火烧烤过的粗糙卵石，表面有黑色烧焦的痕迹。"
  },
  {
    "name": "湿泥",
    "prompt": "混合了石头碎屑和水的深棕色、可塑的泥团。"
  },
  {
    "name": "冒烟的炊具",
    "prompt": "一个粗糙的陶罐，里面盛着水，下面正燃烧着小火。"
  }
]

```

然后玩家解决过的events 就标记为已完成，下次随机选择的时候就不选中这个events.

只有玩家完成了所有的当前所处时代的events，升级到下一个时代，时代顺序为表格顺序：

生存时代 城邦时代  分野时代 帝国时代 理性时代 信仰时代 启蒙时代 全球时代 第二次分野时代  星辰时代/奇点时代

通过用户触发的第二次分野时代的key:【太空电梯】 或 【脑机接口】来决定用户是进入星辰时代还是奇点时代

每次玩家成功解决一个events，就把选择的地块以及随机数量的周围地块 放置一个特殊的标志，标志名称为表格内的“沙盘奖励”，然后从美术资源文件夹里找相同的名字的图片（png,jpg,webp) 来放置，因为放置的标志为2D图片，所以需要下面的地块上的标志遮挡上面的地块的标志。然后永久高亮这一整块的地块的边缘。要呼吸高亮

同时为背景音乐，点击音效，合成音效，events通关音效，卡牌拖动的放置音效，沙盘格子点击音效预留接口
